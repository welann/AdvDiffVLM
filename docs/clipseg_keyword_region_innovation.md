# 基于 CLIPSeg 的关键词区域约束创新方案

## 1. 目标

这份文档把我们刚才讨论的思路整理成一个完整的创新方案，核心目标是：

> 不再使用当前 `AdvDiffVLM` 里基于 GradCAM 的显著性 mask，而是使用“目标图像关键词 -> 目标区域先验 -> 当前去噪图像关键词响应 -> 动态融合 mask”来约束攻击修改范围。

我建议把这条路线命名为：

**KGRC**

即：

**Keyword-Grounded Region Constraint**

如果强调动态融合，也可以写成：

**DKGRC**

即：

**Dynamic Keyword-Grounded Region Constraint**

这条路线的核心不是“换一个生成 mask 的工具”，而是把攻击区域约束从：

- 源图显著性驱动

改成：

- 目标语义驱动 + 当前生成状态自适应驱动

这点是当前 `AdvDiffVLM` 所没有的。

---

## 2. 为什么当前 GradCAM 路线还有改进空间

当前 `AdvDiffVLM` 的 GCMG 路线，核心逻辑是：

1. 用源图 + 真值类别生成 GradCAM
2. 从 CAM 中随机采样热点
3. 在扩散逆过程里只放开局部窗口
4. 其余区域尽量保持和源图 noisy latent 一致

它的问题不是“不能用”，而是它解决的是 **源图显著区域保护**，不是 **目标语义定位**。

换句话说，GradCAM 回答的是：

> 源图哪里重要，最好少改

但 targeted attack 真正更关心的是：

> 目标语义应该优先出现在哪里

这两者不是一回事。

例如：

- 当前源图是一只鸟
- 目标图是一辆红色公交车

那么源图的显著区域可能是鸟本身，而目标语义“bus”在目标图中的主体区域与源图显著区域根本不是同一个语义对象。

所以，如果继续只围绕源图显著性来构造 mask，会有两个问题：

1. 它约束的是“不要破坏源图关键内容”，不是“把攻击集中到目标语义应该出现的地方”
2. 它缺少目标语义的空间模板，因此攻击语义更容易散乱渗透，而不是结构化出现

---

## 3. 为什么不建议直接照搬 `attentionControl.py`

`DiffAttack_v1/attentionControl.py` 这条路线的关键前提是：

- 扩散模型是 **文本条件** 的
- 有显式 token-level cross-attention
- 能直接 hook `CrossAttention.forward`
- 能对指定 token 的注意力图施加 loss 或替换控制

但当前 `AdvDiffVLM` 的主干不是这样：

- 它用的是 `class_label` 条件，而不是 prompt token 条件
- 条件编码器是 `ClassEmbedder`
- 当前主任务里没有可直接使用的“关键词 token -> cross-attention map”

因此，直接照搬 `attentionControl.py` 有两个结构性问题：

1. **条件形式不匹配**
   当前模型没有你要控制的关键词 token 序列。

2. **工程入侵性太强**
   如果为了支持 token attention 而把当前 ImageNet 类条件 LDM 整体换成文本条件模型，那不再是对当前 `AdvDiffVLM` 的小改，而是整个生成底座都变了。

所以更合理的思路不是“在当前模型内部找 token attention”，而是：

> 使用外部的文本分割模型，把关键词转成空间区域先验，再在当前扩散攻击中把这个空间先验用起来。

CLIPSeg 正好满足这个需要。

---

## 4. 创新点到底是什么

这条路线我认为有 4 个明确的创新点。

### 4.1 创新点一：从“源图显著性约束”改成“目标语义区域约束”

当前 GCMG 使用的是源图的 GradCAM，本质上是：

> 用源图 saliency 保护原始结构

而 KGRC 的核心是：

> 用目标图关键词的语义区域先验来约束攻击应该主要作用在哪些空间位置

这意味着攻击范围不再由源图类别决定，而由目标语义决定。

这在 targeted attack 里更自然，也更强。

### 4.2 创新点二：用“静态先验 + 动态观测”做区域融合

不是只对目标图做一次 CLIPSeg 然后得到一个固定 mask，就结束了。

我们还会在扩散逆过程的每一步，基于当前去噪图像再做一次关键词响应估计。

这样最终的区域约束不是静态的，而是：

- 目标图给出先验模板
- 当前图像给出实时观测
- 二者融合出动态约束区域

这比单一静态 mask 更合理，因为攻击过程中语义会逐渐成型。

### 4.3 创新点三：把区域融合写成概率意义上的“先验-观测融合”

这不是简单平均两个 mask，而是可以写成一个很清楚的概率融合模型。

也就是说，这条路线不仅工程上能做，而且数学上也讲得通。

### 4.4 创新点四：在不改当前类条件 LDM 主体的前提下，引入文本语义空间约束

这是很重要的一点。

如果你强行切到 prompt-conditioned diffusion，虽然也能做，但等于把整个底座换掉了。

而 KGRC 的好处是：

- 不需要改掉当前 `AdvDiffVLM` 的类条件 LDM 主体
- 只是在攻击环节外挂一个文本 grounding 分支
- 因而更适合当前代码库的增量式改造

这使它在工程上更现实。

## 5. 方法定义

下面把这套方法完整写成数学形式。

### 5.1 关键词集合

先让 VLM 根据目标图像 `I_{tar}` 生成一段描述文本，再从中提取关键词或短语。

记关键词集合为：

```math
\mathcal{K} = \{(k_1,\alpha_1), (k_2,\alpha_2), \dots, (k_m,\alpha_m)\}
```

其中：

- `k_j` 是第 `j` 个关键词或短语
- `\alpha_j \ge 0`
- `\sum_{j=1}^{m} \alpha_j = 1`

这里的 `\alpha_j` 表示关键词重要性。

建议不要只用单个词，而优先使用名词短语，例如：

- `red bus`
- `brown dog`
- `wooden bench`

因为短语比单词更稳定，语义歧义更小。

---

### 5.2 目标区域先验 `R_j`

设 `S_\theta(I, k)` 表示 CLIPSeg 对图像 `I` 和文本 `k` 输出的 segmentation logit map。

对目标图像 `I_{tar}` 和关键词 `k_j`，先得到：

```math
Z_j^{tar} = S_\theta(I_{tar}, k_j)
```

再经过归一化函数 `\mathcal{N}` 得到概率形式的目标区域先验：

```math
R_j = \mathcal{N}(Z_j^{tar}) \in [0,1]^{H \times W}
```

一个合理的归一化方式是：

```math
\mathcal{N}(Z) =
\frac{\sigma(Z/\tau_{seg})}
{\max(\sigma(Z/\tau_{seg})) + \epsilon}
```

其中：

- `\sigma(\cdot)` 是 sigmoid
- `\tau_{seg}` 是温度系数
- `\epsilon` 是数值稳定项

于是总的目标语义先验为：

```math
R = \sum_{j=1}^{m} \alpha_j R_j
```

这个 `R` 是一个 **静态先验区域图**。

它回答的问题是：

> 目标语义在目标图中主要出现在什么位置。

---

### 5.3 当前步关键词响应图 `A_j^{(t)}`

在扩散逆过程的第 `t` 步，设当前 latent 为 `x_t`，并由采样器得到当前 clean estimate：

```math
\hat{x}_0^{(t)}
```

将其解码得到当前步的图像估计：

```math
\hat{I}_t = D(\hat{x}_0^{(t)})
```

其中 `D(\cdot)` 表示 VQ/VAE decode。

然后对每个关键词 `k_j`，再次使用 CLIPSeg：

```math
Z_j^{(t)} = S_\theta(\hat{I}_t, k_j)
```

归一化得到当前步关键词响应图：

```math
A_j^{(t)} = \mathcal{N}(Z_j^{(t)})
```

再聚合为总响应图：

```math
A^{(t)} = \sum_{j=1}^{m} \alpha_j A_j^{(t)}
```

这个 `A^{(t)}` 是一个 **动态观测图**。

它回答的问题是：

> 在当前去噪图像里，目标关键词正在什么位置开始显现。

---

### 5.4 时间平滑

由于扩散逆过程中的图像语义是逐步形成的，所以 `A^{(t)}` 会有较强抖动。

因此建议引入 EMA 平滑：

```math
\bar{A}^{(t)} = \rho \bar{A}^{(t+1)} + (1-\rho) A^{(t)}
```

其中：

- `\rho \in [0,1)`
- 因为是逆过程，所以这里可以用 `t+1` 表示前一个更噪的状态

为了简洁，也可以写成：

```math
\bar{A}_t = \rho \bar{A}_{t-1} + (1-\rho)A_t
```

只要实现时序一致即可。

这样得到的是一个稳定得多的观测响应图。

---

## 6. 核心创新：把区域融合写成概率融合，而不是简单平均

这是我认为这条路线里最值得强调的数学创新点。

### 6.1 为什么不推荐直接平均

最简单的融合当然可以写成：

```math
M_t = \lambda_t R + (1-\lambda_t)\bar{A}^{(t)}
```

它能用，但缺点是：

- 它只是经验上的 convex combination
- 没有明确的概率解释
- 当 `R` 和 `\bar{A}^{(t)}` 都非常接近 0 或 1 时，线性平均会过于平滑

### 6.2 更好的方式：logit 空间下的加权 Product-of-Experts

把 `R` 和 `\bar{A}^{(t)}` 都看成像素级 Bernoulli 概率图。

定义：

```math
\mathrm{logit}(p)=\log\frac{p}{1-p}
```

那么可以把目标先验和当前观测视作两个 expert，在 logit 空间做融合：

```math
M_t =
\sigma\left(
\lambda_t \,\mathrm{logit}(R)
+
(1-\lambda_t)\,\mathrm{logit}(\bar{A}^{(t)})
\right)
```

其中：

- `M_t` 是最终的动态区域约束图
- `\lambda_t \in [0,1]`

### 6.3 为什么这个公式合理

如果把先验和观测看成条件独立的证据源，那么像素级 posterior 的 log-odds 会近似变成多个证据 log-odds 的和。

因此，上面的式子可以理解为一个 **加权的 Product-of-Experts 融合**。

直觉上：

- `R` 提供“目标语义大概应该在哪”的模板
- `\bar{A}^{(t)}` 提供“当前生成里语义实际在哪里出现”的观测
- `M_t` 是二者融合后的后验区域图

这比简单平均更有解释性，也更适合写成论文里的创新点。

---

## 7. `\lambda_t` 如何设置

`R` 和 `\bar{A}^{(t)}` 的可信度在不同时间步并不一样。

### 7.1 基本直觉

在逆过程早期：

- 当前图像还比较模糊
- CLIPSeg 在 `\hat{I}_t` 上得到的响应可能不稳定
- 应该更信任目标先验 `R`

在逆过程后期：

- 当前图像语义更成型
- `\bar{A}^{(t)}` 更有意义
- 应该逐步提高观测项权重

### 7.2 一个可行的时间调度

定义归一化进度：

```math
p_t = 1 - \frac{t}{t^*}
```

其中：

- `t^*` 是开始启用攻击/区域约束的起始步
- `p_t \in [0,1]`

然后设：

```math
\lambda_t =
\lambda_{min}
+
(\lambda_{max}-\lambda_{min}) e^{-\beta p_t}
```

这样：

- 早期 `p_t \approx 0`，`lambda_t \approx lambda_max`
- 后期 `p_t \to 1`，`lambda_t` 逐渐向 `lambda_min` 靠近

例如可以取：

- `\lambda_{max}=0.85`
- `\lambda_{min}=0.35`

这就实现了：

> 早期更依赖目标先验，后期更依赖当前图像响应

## 8. 如何把 `M_t` 用到攻击里

这里有两条路线。

### 8.1 路线一：作为 soft gate 约束攻击梯度

设原始 targeted gradient 为：

```math
g_t = \nabla_{x_t} \mathcal{L}_{target}^{(t)}
```

则用区域约束图门控后：

```math
g_t' = M_t^\nu \odot g_t
```

其中：

- `\nu > 0` 控制门控强度
- `\odot` 表示逐元素乘法

然后再做 latent 更新：

```math
x_t \leftarrow x_t + s g_t'
```

这种方式简单、稳定，而且不要求 CLIPSeg 参与反向传播。

这条路线最适合第一版实现。

### 8.2 路线二：作为动态 mask 融合源图 noisy latent 与 adversarial latent

延续当前 GCMG 的结构，可以写成：

```math
\hat{x}_t = (1-M_t)\odot x_t^{orig} + M_t \odot x_t^{adv}
```

其中：

- `x_t^{orig}` 是源图在当前步的 noisy latent
- `x_t^{adv}` 是当前攻击生成 latent

这表示：

- `M_t` 高的区域，允许更多攻击修改
- `M_t` 低的区域，更偏向保留原始结构

这和当前 `AdvDiffVLM` 的 `img_orig * mask + (1-mask) * img` 结构是兼容的，只需要把现在的 `cam` 派生 mask 换成新的 `M_t`。

## 9. 再进一步：可以加一个区域一致性正则项

如果你后续愿意让 CLIPSeg 参与更强的约束，还可以在损失里加入区域一致性项。

### 9.1 Dice 形式

对每个关键词 `k_j`，定义：

```math
\mathcal{L}_{loc,j}^{(t)}
=
1 -
\frac{2\langle A_j^{(t)}, R_j\rangle + \epsilon}
{\|A_j^{(t)}\|_1 + \|R_j\|_1 + \epsilon}
```

总区域损失为：

```math
\mathcal{L}_{loc}^{(t)}
=
\sum_{j=1}^{m}\alpha_j \mathcal{L}_{loc,j}^{(t)}
```

然后总损失：

```math
\mathcal{L}^{(t)}
=
\mathcal{L}_{target}^{(t)}
 \beta_{loc}\mathcal{L}_{loc}^{(t)}
```

### 9.2 这个损失的含义

它要求：

- 当前图像里的关键词响应区域
- 不要只是“哪里有都行”
- 而是尽量和目标图像里该关键词的区域结构一致

也就是说，你约束的不只是“目标语义出现”，而是“目标语义在合适的位置出现”。

这比当前 GradCAM 路线更精细。

## 10. 为什么这条路线是走得通的

这一节专门回答“为什么它可行”。

### 10.1 理由一：它绕开了当前模型没有 token cross-attention 的结构障碍

这是最现实的原因。

当前 `AdvDiffVLM` 不是 prompt-conditioned diffusion，所以无法像 `attentionControl.py` 那样直接读某几个词的 cross-attention。

CLIPSeg 提供了一个架构外的解决方式：

- 不依赖当前扩散模型内部是否有文本 token
- 只要你能拿到一张图像和一个文本短语
- 就能得到对应的语义区域图

因此，这条路线能在当前代码库里以增量方式接入。

### 10.2 理由二：它把攻击区域约束从“源图保护”改成“目标语义定位”

当前 GCMG 的本质是：

- 少破坏源图重要区域

KGRC 的本质是：

- 把攻击修改集中到目标语义对应区域

对于 targeted attack 来说，后者显然更直接、更合理。

### 10.3 理由三：它具有“先验-观测”双重鲁棒性

如果只用目标图 mask：

- 太静态
- 无法反映当前生成状态

如果只用当前步响应图：

- 太依赖当前图像质量
- 早期 timestep 会不稳定

把两者融合之后：

- `R` 保证目标语义不会漂离正确空间模板
- `A^{(t)}` 让 mask 能跟着当前图像逐步修正

这就是它能稳定工作的根本原因。

### 10.4 理由四：CLIPSeg 与关键词短语天然匹配

你现在的想法里有一个很重要的操作：

- 让 VLM 给目标图像生成文本
- 并指定关键短语

而 CLIPSeg 正好就是：

- 图像 + 文本短语
- 输出相应区域图

所以这两步在语义接口上是自然匹配的，不是强行拼凑。

### 10.5 理由五：它还能兼容当前 AEGE 与多轮外循环

这点也很重要。

KGRC 不会破坏当前 `AdvDiffVLM` 的主框架：

- AEGE 仍然负责目标语义梯度估计
- 多轮 `N` 次外循环仍然可以保留
- 改的只是“攻击梯度在哪里释放、哪里保守”

也就是说，这是一种与当前算法 **正交可组合** 的创新，而不是替代掉整个方法。

---

## 11. 相比当前方法，这条路线的增量创新总结

可以把创新压缩成下面 4 句话：

1. 当前方法用的是 **源图显著性约束**，新方法改成 **目标关键词区域约束**。
2. 当前方法的 mask 是 **静态源图驱动**，新方法的 mask 是 **目标先验 + 当前观测动态融合**。
3. 当前方法的 mask 没有明确概率解释，新方法可写成 **加权 PoE 后验融合**。
4. 当前方法只能表达“哪里少改”，新方法还能表达“目标语义应当主要在哪里出现”。

这 4 点足够构成一个独立方法改进方向。

---

## 12. 推荐的落地顺序

如果你真的要做，我建议分三步，而不是一步做满。

### 第一步：最小可行版

只做：

- 目标图 `R`
- 当前步响应 `A^{(t)}`
- 简单时间平滑
- 线性融合 `M_t`
- 用 `M_t` 门控攻击梯度

这是最稳、改动最小的版本。

### 第二步：升级到 logit 空间融合

把：

```math
M_t = \lambda_t R + (1-\lambda_t)\bar{A}^{(t)}
```

升级成：

```math
M_t =
\sigma\left(
\lambda_t \,\mathrm{logit}(R)
+
(1-\lambda_t)\,\mathrm{logit}(\bar{A}^{(t)})
\right)
```

这样数学解释更完整。

### 第三步：再加入区域一致性损失

如果想进一步提升“目标语义在正确位置出现”的约束强度，再引入：

```math
\mathcal{L}_{loc}^{(t)}
```

---

## 13. 一句话总结

这条路线的真正创新，不是“把 GradCAM 换成 CLIPSeg”这么简单，而是：

> 用目标图关键词生成的区域先验来定义“目标语义应当在哪里出现”，再用当前去噪图像的关键词响应作为动态观测，通过概率意义上的先验-观测融合构造时变区域约束，从而把当前 `AdvDiffVLM` 的攻击区域控制从源图显著性保护，升级为目标语义引导的结构化攻击范围控制。

如果后续你要把它写进论文或方法设计里，这个表述是足够成立的。
